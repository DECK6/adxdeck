OSMU 콘텐츠 자동화 수업 정리 노트

이 노트는 2026년 7월 5일 현장 수업 녹음 내용만 바탕으로 정리한 복습용 노트입니다. 미리 짜인 진행표가 아니라, 수업 중 실제로 나온 설명, 실습, 질문, 오류, 해결법, 결과물 피드백을 중심으로 정리했습니다.

개인 이름과 민감한 업무 맥락은 공유용으로 일반화했습니다.

1. 수업의 핵심 메시지

수업에서 반복해서 강조된 핵심은 다음입니다.

OSMU는 하나의 큰 자동화 버튼이 아니라, 리서치 → 글감 → 블로그 → 카드뉴스 → 숏폼을 단계별로 나누고 다시 연결하는 방식입니다.

처음부터 모든 것을 한 번에 자동화하려고 하면 막히기 쉽습니다. 수업에서는 아래처럼 나누어 생각하라고 설명했습니다.

  • 블로그 자동화만 따로 할 수 있습니다.
  • 카드뉴스 이미지만 따로 자동화할 수 있습니다.
  • 영상 자동화만 따로 떼어 연습할 수 있습니다.
  • 이미 가지고 있는 파일이나 글감이 있으면 그것을 원천 자료로 삼을 수 있습니다.
  • 아직 아이템이 없으면 트렌드 리서치부터 시작할 수 있습니다.

수업 후 혼자 다시 할 때도 “전체 자동화”보다 내가 지금 어느 단계에서 막혔는지를 먼저 확인하는 편이 좋습니다.

계획된 순서대로 매끈하게 흘러간 수업이라기보다, 현장 설치와 참가자별 문제 해결을 함께 진행한 실습형 워크숍에 가까웠습니다.

3. 참가자들이 실제로 가져온 고민

수업에서 나온 고민은 모두 조금씩 달랐습니다. 아래는 개인 정보를 빼고 유형으로 정리한 것입니다.

유형 실제 고민 수업에서 나온 방향
블로그 자동화 경험자 블로그 오케스트레이션 이후 카드뉴스와 숏폼까지 확장하고 싶음 블로그, 카드뉴스, 숏폼을 한꺼번에 보지 말고 모듈별로 분리
패션/브랜드 90년대·올드스쿨 무드와 최신 트렌드를 엮어 콘텐츠화하고 싶음 과거 지식은 지식베이스에 쌓고, 최신 트렌드 검색과 매칭
전시·컨벤션/MICE 현장에서 본 행사와 운영 경험을 자기 관점으로 포트폴리오화하고 싶음 내부 경험과 현장 인사이트를 콘텐츠의 차별점으로 사용
의료·건강 콘텐츠 환자들이 관심 가질 신약, 치료 트렌드, 희망 콘텐츠를 만들고 싶음 단발 정보가 아니라 다음 콘텐츠와 IP로 이어지게 설계
프랜차이즈 SaaS 업계 이슈와 자사 가치 제안을 연결해 검색·AI 노출을 높이고 싶음 뉴스, 유튜브, 커뮤니티, 카페를 소스화하고 자사 소개서와 매칭
교육 영상 학교폭력 예방 등 5분 이상 교육용 영상을 만들고 싶음 캐릭터 시트와 배경 레퍼런스를 먼저 만들어 장면 단위로 이어붙이기
개인 지식관리 카카오톡, 드라이브, Obsidian 자료를 연결해 제안서와 관계도를 만들고 싶음 내부 자료를 읽고 맥락을 정리하는 개인 지식 자동화로 확장

수업의 중요한 지점은 “모두 같은 결과물을 만들자”가 아니라, 각자 가진 도메인 지식과 자료를 어떻게 자동화 흐름에 넣을지였습니다.

insane-search X/Twitter, 네이버 블로그, 쿠팡, 인스타, 틱톡 등 일반 검색으로 어려운 공개 자료 탐색 보조

도구를 많이 아는 것보다 중요한 것은 어떤 도구가 어느 단계의 문제를 해결하는지 구분하는 것입니다.

5. 설치와 환경 세팅에서 실제로 막힌 지점

수업 초반에는 설치와 로그인 문제가 많이 나왔습니다.

자주 막힌 문제

  • Claude Code와 Codex를 어디에서 실행해야 하는지 헷갈림
  • Git, Node, GitHub 로그인이 안 되어 있음
  • Windows와 Mac의 설치 방식이 다름
  • 앱을 종료했다가 다시 켜야 설치가 반영되는 경우가 있음
  • 에이전트가 작업 중인지 멈춘 것인지 판단하기 어려움
  • 인증 키 또는 접근 권한 오류가 나옴
  • Pixverse 코드 등록 위치를 찾기 어려움
  • 설치와 생성 작업을 한 대화에서 많이 해서 세션 사용량이 빨리 참

수업 중 나온 해결법

이 에러 메시지를 읽고, 지금 무엇이 문제인지와 다음 조치만 단계별로 알려줘.

지금 작업이 진행 중인지 확인해줘. 멈춰 있다면 어디에서 멈췄는지 알려줘.

필요한 의존성이 있으면 설치까지 진행해줘. 설치 전 확인이 필요한 항목만 나에게 물어봐.

설치가 길어질 때는 계속 새 명령을 넣기보다, 현재 실행 중인 작업이 있는지 먼저 확인해야 합니다.

6. 리서치 자동화에서 기억할 점

수업에서는 리서치 자동화가 단순 검색이 아니라고 설명했습니다. 특히 검색이 막히는 사이트나 로그인 필요한 페이지가 많기 때문에, 에이전트가 실패했을 때 내용을 지어내지 않도록 조심해야 합니다.

핵심 포인트

  • 검색 실패를 인정하지 않고 내용을 지어내는 도구가 있을 수 있습니다.
  • X/Twitter, 네이버, 쿠팡, 인스타, 틱톡 등은 일반 검색 접근이 제한될 수 있습니다.
  • insane-search 같은 도구를 명시적으로 사용하라고 지시해야 할 때가 있습니다.
  • 커뮤니티 반응은 트렌드 감지에는 좋지만, 사실 검증 근거로는 약할 수 있습니다.
  • 여러 에이전트가 조사하고, 다른 에이전트가 리뷰하게 하면 할루시네이션을 줄일 수 있습니다.

수업 중 나온 지시 방식

Insane Search를 이용해서 이 주제의 최신 트렌드를 찾아줘.

뉴스뿐 아니라 유튜브, 커뮤니티, 네이버 블로그/카페에서 사람들이 실제로 어떤 말을 하는지 찾아줘.

검색 결과를 사실, 의견, 추정, 콘텐츠 소재로 나눠서 정리해줘.

7. 블로그 자동화에서 기억할 점

블로그 파트에서는 “AI가 글을 길게 써준다”보다 카테고리별 문법과 사람의 후킹 감각이 중요하다고 설명했습니다.

수업에서 나온 블로그 포인트

  • 원천 글감이나 리서치 결과를 기반으로 블로그 글을 만듭니다.
  • 간단히 “대시보드 관련 글”처럼 메모만 던져도 에이전트가 조사해서 살을 붙일 수 있습니다.
  • 원하는 문체의 예시 블로그를 Markdown 파일로 저장해 참고하게 할 수 있습니다.
  • 하지만 예시를 주면 제목과 구조까지 너무 비슷하게 베끼는 문제가 생길 수 있습니다.
  • 제목과 후킹은 아직 사람이 직접 검수하는 편이 좋습니다.
  • 네이버 블로그는 카테고리마다 잘 먹히는 제목, 키워드 밀도, 소제목 구조가 다릅니다.
  • 상위 노출 글을 모아 공통 규칙을 뽑아내면 더 나은 초안을 만들 수 있습니다.

예시를 줄 때 꼭 붙일 말

이 예시는 문체와 구성 참고용이야.
제목, 문장 구조, 비유를 그대로 따라 하지 말고 새롭게 써줘.

제목은 예시와 다르게 만들어줘.
검색 의도와 후킹을 살리되, 똑같은 패턴을 반복하지 마.

8. Skill MD를 저장하고 다듬는 방식

수업 중에는 dexa.art/osmu에 있는 범용 Skill MD를 복사해서 Claude Code나 Codex에 넣고, “스킬로 저장해줘”라고 시키는 흐름이 나왔습니다.

중요한 것은 스킬을 한 번 저장하고 끝내는 것이 아니라, 결과물을 보면서 계속 수정하는 것입니다.

수업 중 나온 스킬 수정 예시

앞으로 결과물에는 한자가 나오지 않게 스킬을 수정해줘.

카드뉴스와 숏폼 이미지의 컬러톤을 한 가지로 통일하도록 스킬에 명시해줘.

로고와 글씨체, 브랜드 톤을 앞으로 모든 이미지에 반영하도록 스킬을 수정해줘.

카드뉴스는 필요 없고, 숏폼만 만들도록 스킬을 바꿔줘.

숏폼 대신 롱폼 또는 스토리형 영상 제작에 맞게 스킬을 변형해줘.

스킬은 “한 번 만든 프롬프트”가 아니라, 내 작업 방식이 쌓이는 운영 문서에 가깝습니다.

9. Pixverse와 영상 생성에서 나온 현실적인 이야기

수업에서는 Pixverse와 영상 모델에 대한 현실적인 비용과 품질 이야기도 많이 나왔습니다.

기억할 점

  • 제공받은 코드는 아끼기보다 실습 중 충분히 써보는 것이 좋다고 안내했습니다.
  • Pixverse 기본 모델이 항상 최고 품질을 내는 것은 아닙니다.
  • Pro와 Pro Plus는 사용할 수 있는 모델 범위가 다를 수 있습니다.
  • 외부 고품질 영상 모델은 10-15초 영상 하나에도 크레딧이 크게 소모될 수 있습니다.
  • 좋은 모델을 써도 한 번에 완벽한 결과가 나오는 것은 아닙니다.
  • AI 영상 수정은 공짜가 아닙니다. 장면을 다시 만들면 시간과 크레딧이 다시 듭니다.

Pixverse를 쓰게 하고 싶을 때

Pixverse를 이용해서 이미지와 영상을 만들어줘.
Python이나 단순 이미지 조합으로 대체하지 말고, Pixverse 생성 흐름을 사용해줘.

10. 결과물이 이상할 때 고치는 법

문제 수업에서 나온 원인 고치는 방향
한자가 들어감 해외 자료나 중국어 이름이 그대로 들어옴 한자 사용 금지를 스킬에 명시
이미지 색이 섞임 컬러톤 규칙이 없음 한 가지 컬러톤과 폰트 규칙을 스킬에 추가
카드뉴스와 숏폼이 따로 놈 카드뉴스 결과를 숏폼 장면이 참조하지 않음 카드뉴스 이미지/메시지를 기준으로 숏폼을 다시 만들라고 지시
예시와 너무 똑같음 예시를 스타일 참고가 아니라 복제 대상으로 사용 예시 복제 금지, 제목 다양화 지시
결과물 순서가 이상함 스킬이 두 번 실행됐거나 백그라운드 작업이 남음 실행 중인 작업을 먼저 확인하고 하나만 다시 실행
세션이 찼음 설치와 생성에서 토큰을 많이 씀 설치용 대화와 생성용 대화를 분리
이미지가 예쁜데 내용과 안 맞음 브랜드 톤과 메시지의 연결이 약함 레퍼런스 이미지와 메시지 기준을 다시 제시
자막 위치/줄바꿈이 어색함 영상 후편집 규칙이 부족함 자막 위치, 줄 수, 문장 길이를 별도 규칙으로 지정

11. 숏폼과 긴 영상 제작 팁

수업 후반부에는 숏폼을 넘어 긴 영상과 교육 영상 제작에 대한 질문도 나왔습니다.

짧은 숏폼

  • 블로그 전체를 넣으려고 하지 말고 한 가지 각도만 잡습니다.
  • 카드뉴스에서 만든 이미지나 메시지를 재사용하면 일관성이 좋아집니다.
  • 내레이션과 화면이 따로 놀면 다시 구성해야 합니다.
  • 자막 위치와 줄바꿈은 반드시 확인해야 합니다.

일부 장면만 다시 만들기

전체를 다시 만들지 않고 특정 장면만 교체할 수 있습니다.

세 번째 영상만 다시 만들어서, 기존 영상과 다시 조립해줘.

긴 영상과 교육 영상

긴 영상은 한 번에 길게 생성하는 것이 아니라, 짧은 장면을 여러 개 만들어 이어붙이는 방식으로 설명되었습니다.

  • 캐릭터 시트를 먼저 만듭니다.
  • 배경과 로케이션 이미지를 따로 만듭니다.
  • 장면별 이미지를 만들고 영상화합니다.
  • 복장이 바뀌면 새 캐릭터 시트를 만듭니다.
  • 같은 캐릭터와 배경을 반복 참조해야 일관성이 유지됩니다.

강사는 이것을 영화 제작에 비유했습니다. 배우와 로케이션을 먼저 정하듯이, AI 영상에서도 캐릭터와 배경을 먼저 확보해야 합니다.

12. TTS와 목소리

수업에서는 영상 품질에서 음성도 중요하다고 이야기했습니다.

  • Edge TTS는 무료에 가깝게 쓸 수 있는 기본 음성 후보로 소개되었습니다.
  • 더 자연스러운 음성을 원하면 ElevenLabs 같은 유료 서비스를 고려할 수 있습니다.
  • 목소리를 학습시킬 때 녹음 품질이 낮으면 잡음의 질감도 같이 섞일 수 있습니다.
  • 한국어 여성 목소리처럼 데이터가 적은 영역은 목소리 마켓에서 수요가 있을 수 있다는 이야기도 나왔습니다.

숏폼에서는 영상만큼이나 자막과 목소리의 신뢰감이 중요합니다.

13. 수업 후 혼자 다시 해볼 순서

현장 녹음에서 나온 흐름을 기준으로, 수업 후에는 아래 순서로 다시 해보면 좋습니다.

  1. 내가 자동화하고 싶은 주제 하나를 고릅니다.
  2. 주제가 없으면 트렌드 리서치부터 시작합니다.
  3. 주제가 있으면 기존 파일, 글, 자료를 원천 자료로 넣습니다.
  4. 리서치 결과를 사실, 해석, 콘텐츠 소재로 나눕니다.
  5. 블로그 글 구조를 먼저 만듭니다.
  6. 카드뉴스 5-7장 구성으로 쪼갭니다.
  7. 숏폼 대본과 장면 구성을 만듭니다.
  8. Pixverse 등 영상 도구로 일부 장면을 생성합니다.
  9. 결과물을 보고 색, 자막, 로고, 이미지 톤, 내용 적합성을 검수합니다.
  10. 잘된 점과 문제점을 스킬에 반영합니다.

한 번에 완성하려고 하기보다, 한 번 실행하고 → 결과를 보고 → 스킬을 고치는 반복이 중요합니다.

14. 수업에서 바로 가져갈 수 있는 말들

아래 문장들은 수업 중 나온 문제 해결 방식에 맞춰 다시 사용할 수 있는 지시문입니다.

이 결과물이 내용과 맞는지 평가해줘. 예쁘지만 메시지와 어긋나는 부분이 있으면 알려줘.

카드뉴스와 숏폼이 같은 원천 자료에서 나온 것처럼 보이도록 톤과 메시지를 맞춰줘.

지금 결과물에서 다시 만들 필요가 있는 장면만 골라줘. 전체 재생성은 피하고 싶어.

내가 준 예시를 그대로 베끼지 말고, 문체의 장점만 참고해서 새 글로 써줘.

이번 결과에서 스킬에 반영해야 할 규칙을 정리해줘.

15. 조심해야 할 점

  • AI가 리서치한 내용을 바로 사실로 믿지 않습니다.
  • 공개 자료와 사용 권한이 있는 자료를 우선 사용합니다.
  • 개인정보, 고객 정보, 의료 정보, 상담 내용, 회사 내부 자료는 함부로 넣지 않습니다.
  • 저작권 있는 글, 이미지, 음악, 영화, 책 내용을 그대로 복제하지 않습니다.
  • “가능성”, “계획”, “추진”을 “확정”처럼 말하지 않습니다.
  • 예쁜 결과물보다 내용과 맞는 결과물이 더 중요합니다.
  • 자동화는 사람의 판단을 없애는 것이 아니라, 반복 작업을 줄여 판단할 시간을 확보하는 일입니다.

16. 마지막 정리

수업에서 가장 많이 반복된 원리는 간단합니다.

  1. 내가 가진 주제나 자료를 정합니다.
  2. 리서치로 근거를 보강합니다.
  3. 원천 글감을 하나로 정리합니다.
  4. 블로그, 카드뉴스, 숏폼으로 나눠 변환합니다.
  5. 결과를 보고 스킬을 고칩니다.
  6. 다음에는 더 적은 설명으로 더 나은 결과가 나오게 만듭니다.
