0. 수업 방향

이 파일 하나가 교안입니다. 외부 패키지나 별도 코드 폴더를 전제로 하지 않습니다. 각 Lab 카드 안에 프롬프트, Arduino 코드, 수동 SVG 회로도/핀맵, TouchDesigner 입력확인(가능시), PASS/HOLD/FAIL 기준을 함께 배치했습니다.

이 자료는 “예제 코드 따라하기”가 아니라, 학생이 실제 키트 부품을 보고 전기 흐름을 이해하고, AI agent가 만든 답을 회로 evidence로 검증하는 수업을 목표로 합니다.

AI는 페어 엔지니어

AI agent는 회로 초안, 코드 초안, 오류 질문 정리, 발표 자료 작성을 돕습니다. 하지만 부품 정격, 핀 순서, 극성, 전원 안전, 실제 작동 여부는 사람이 확인합니다.

키트 부품 중심

DHT/PIR/NeoPixel/servo처럼 키트에 없는 부품은 필수 경로에서 제외했습니다. 대신 NTC, CDS, IR, 조이스틱, HC-SR04, MPU6050, RTC, LCD/OLED, 74HC595, MAX7219, L293D, relay, HC-05를 중심으로 수업을 설계했습니다.

구동부 안전 경계

fan, relay, transistor, motor driver는 Arduino pin으로 직접 부하를 감당하지 않습니다. relay는 AC 전원을 다루지 않고, LED dummy load로만 contact 개념을 설명합니다.

0-1. 처음 만지는 학생을 위한 친절 가이드

이 섹션은 강사가 수업 시작 직후 그대로 읽어도 되는 안내입니다. 학생이 “어디부터 봐야 하는지”를 잃지 않도록, 모든 실습을 같은 순서로 반복하게 만듭니다.

강사용 첫 멘트. “오늘 목표는 천재처럼 코드를 짜는 게 아닙니다. 전기가 어디서 나가고 어디로 돌아오는지, 센서값이 어떤 숫자로 들어오는지, AI가 만든 코드를 우리가 어떤 증거로 검증하는지 배우는 것입니다. 실패하면 망한 게 아니라 evidence가 하나 생긴 겁니다.”

학생이 오늘 가져갈 한 문장

전원은 힘, GND는 돌아오는 길, Signal은 정보, Serial은 증거입니다. 회로가 안 되면 이 네 가지를 순서대로 확인합니다.

회로 카드 읽는 법

  1. 부품 이름을 먼저 찾습니다.
  2. +5V/VCC 빨간 선을 찾습니다.
  3. GND 검정 선이 어디로 돌아오는지 봅니다.
  4. Arduino pin으로 가는 signal/PWM 선을 찾습니다.
  5. Serial Monitor에 어떤 값이 떠야 하는지 확인합니다.

AI에게 맡기면 안 되는 것

AI는 코드와 설명을 도와주지만, 실제 부품 방향, 저항값, 전원 연결, GND 공통, 발열·냄새·소리 같은 안전 신호는 사람이 확인합니다.

처음 실패했을 때의 기준

한 번에 성공하지 않아도 됩니다. 컴파일 성공 → 업로드 성공 → Serial 값 확인 → 배선 확인 → TD 확인 순서로 어디에서 멈췄는지만 찾으면 됩니다.

모든 Lab의 기본 진행 순서

1. 부품 찾기실물 키트에서 부품을 먼저 손으로 찾고, 모듈 pin label을 읽습니다.

4. 코드 업로드Arduino IDE에서 보드/포트를 확인하고 예제 코드를 올립니다.
5. Serial evidenceSerial Monitor에 기대한 JSON field가 나오는지 봅니다.

1. 먼저 설명해야 할 핵심 개념

학생에게 실습 전 반복해서 보여줄 개념입니다. 각 Lab은 이 개념 중 하나 이상을 반드시 회수합니다.

Microcontroller

Arduino UNO R3는 작은 PC가 아니라 센서 값을 읽고 정해진 코드를 반복 실행해 핀에 전기 신호를 내보내는 제어기입니다.

전기 흐름과 GND

회로는 전류가 돌아오는 길이 있어야 합니다. 외부 전원, driver, 통신 모듈을 연결할 때 common GND가 필요한 이유를 실제 회로에서 계속 확인합니다.

Digital / Analog / PWM

Digital은 HIGH/LOW, Analog input은 0–1023 ADC, PWM은 실제 analog voltage가 아니라 빠른 ON/OFF 비율로 만든 output command입니다.

Raw → Calibrated → State

센서 raw value는 곧 의미가 아닙니다. baseline, threshold, smoothing/debounce, state naming을 거쳐 작품의 input cue가 됩니다.

Serial Evidence

Serial Monitor와 arduino-cli monitor는 “보드 내부를 보는 창”입니다. 학생은 작동했다고 말하기 전에 raw log와 state log를 남깁니다.

Command와 Physical Success

relay input이 켜진 것, fan PWM command가 나간 것, Serial ACK가 온 것은 물리 성공과 다릅니다. 관찰·readback·안전 확인이 필요합니다.

2. 구성품 설명을 풍부하게 보기

각 부품을 “무엇인가”에서 끝내지 않고, 수업에서 어떤 개념을 맡는지까지 설명합니다.

구성품 인식은 실물 키트 우선. 이전 생성 이미지는 배선/부품 형태가 부정확할 수 있어 제거했습니다. 수업에서는 실제 키트를 펼쳐 놓고 UNO, breadboard rail, 저항값, LED 극성, 센서/모듈 pin label을 직접 확인한 뒤 아래 수동 SVG와 핀맵을 대조합니다.

Arduino UNO R3

5V 기준의 수업 보드입니다. digital pin, analog input, PWM pin, USB Serial의 차이를 UNO 위에서 직접 짚습니다. D0/D1은 USB Serial과 겹치므로 초급 회로에서는 피하고, I2C는 A4/A5를 씁니다.

브레드보드와 점퍼선

브레드보드는 내부 연결이 숨어 있는 판입니다. 전원 rail이 중간에서 끊기는 모델인지, 중앙 홈을 기준으로 양쪽이 분리되는지 먼저 시연합니다. 빨강=5V, 검정=GND처럼 색상 규칙을 정하면 오류가 줄어듭니다.

저항 카드와 저항 세트

LED에는 220Ω/330Ω을 기본으로 두고, CDS/NTC 분압에는 10KΩ을 기준으로 씁니다. 100KΩ/1MΩ은 floating-like behavior와 noise를 보여줄 때만 제한적으로 씁니다.

택트 스위치

버튼은 간단해 보여도 floating, pull-up, active-low, debounce를 모두 설명할 수 있는 핵심 부품입니다. 수업에서는 항상 raw HIGH/LOW와 pressed boolean을 따로 출력하게 합니다.

CDS, NTC, 가변저항

세 부품 모두 analog input 교육의 중심입니다. 학생은 raw value를 바로 의미로 쓰지 않고, 현장 baseline과 threshold, normalization 과정을 거쳐 state로 올리는 연습을 합니다.

HC-SR04, IR, 조이스틱

거리·원격·2축 입력을 담당합니다. 이 세 부품은 단일 센서값보다 mode, zone, cursor, key map처럼 “입력을 의미 있는 인터랙션 상태로 번역하는 법”을 가르치기에 좋습니다.

LCD/OLED/RTC/MPU6050

I2C bus 묶음입니다. scanner로 주소를 찾고, 각 모듈에서 raw/read를 확인한 뒤 display dashboard로 합칩니다. 주소 발견만으로 PASS 처리하지 않습니다.

74HC595/MAX7219

Arduino pin이 부족해지는 순간의 확장 전략입니다. 74HC595는 shift register의 원리를, MAX7219는 display driver가 왜 필요한지 보여줍니다.

L293D, 팬, 릴레이, 트랜지스터

구동부는 안전 경계를 분명히 둡니다. Arduino pin은 명령을 보내고, 실제 부하는 driver/relay/transistor와 별도 전원 쪽에서 처리합니다. command success와 physical success를 분리합니다.

HC-05, CP2102, RS232/RS485

Serial protocol의 확장입니다. TTL, USB, Bluetooth, RS232/RS485는 “메시지 형식”과 “전기 레벨/물리 계층”이 다르다는 점을 구분하게 합니다.

3. 6시간 집중 운영안 — 왜 이 순서로 가는가

기존 시간표가 “무엇을 한다”만 보여줬다면, 이 보강판은 각 시간대마다 왜 이 활동이 필요한지, 강사가 무엇을 설명해야 하는지, 학생이 무엇을 증거로 남겨야 하는지까지 보여줍니다. 초보자는 부품 이름보다 먼저 “전원·GND·신호·증거” 흐름을 반복해서 몸에 익혀야 합니다.

운영 원칙. 매 블록은 ① 강사 5분 개념 설명 ② 실물 부품 찾기 ③ 회로/코드 실행 ④ Serial evidence 확인 ⑤ PASS/HOLD/FAIL 기록 순서로 닫습니다. 시간표는 많은 것을 끝내기 위한 압박표가 아니라, 어느 단계에서 학생이 막혔는지 찾기 위한 지도입니다.
0:00–0:25

키트 맵과 개발환경

UNO, USB, 브레드보드, 저항, 점퍼를 실제로 꺼내 분류하고 Arduino IDE upload와 CLI compile/upload/monitor 흐름을 확인합니다.

왜 먼저 하나요? 학생이 부품을 못 알아보면 회로도도 코드도 따라갈 수 없습니다. 이 시간은 “수업 속도”가 아니라 “실물 인식과 개발환경 불안 제거”를 위한 구간입니다.
강사 설명

UNO는 작은 컴퓨터가 아니라 핀을 읽고 쓰는 제어기라는 점, USB는 전원+업로드+Serial 창이라는 점을 설명합니다.

학생 활동

부품을 가족별로 나누고, 보드/포트 선택 후 Blink 또는 빈 sketch 업로드를 해봅니다.

Evidence

IDE 업로드 성공 화면, Serial Monitor 또는 CLI monitor가 열리는 화면, 부품 분류 사진/메모.

넘어가기 조건

학생이 자신의 UNO, USB, 브레드보드 rail, 220Ω/330Ω 저항을 구분할 수 있으면 다음 단계로 갑니다.

0:25–1:10

초급 디지털

LED, 버튼, 부저로 전류 제한, GND return, INPUT_PULLUP, active-low, debounce를 실습합니다.

왜 디지털부터 하나요? HIGH/LOW는 센서·통신·출력의 가장 작은 언어입니다. LED와 버튼을 이해하면 이후 모든 회로의 “켜짐/꺼짐, 눌림/안 눌림, 성공/실패”를 말할 수 있습니다.
강사 설명

LED에는 저항이 필요하고, 버튼은 그냥 누르는 부품이 아니라 floating과 pull-up을 배우는 부품이라고 설명합니다.

학생 활동

B1 LED, B2 버튼, B3 부저를 빠르게 돌며 각 회로의 GND return을 손으로 따라갑니다.

Evidence

LED on/off, 버튼 raw/pressed JSON, 부저 tone 또는 beep log.

넘어가기 조건

“버튼을 누르면 LOW가 되는 이유”와 “저항 없는 LED가 왜 위험한지”를 말로 설명하면 통과입니다.

1:10–2:00

초급 아날로그

pot, CDS, NTC, joystick으로 voltage divider, raw value, calibration, deadzone을 익힙니다.

왜 아날로그를 따로 묶나요? 센서값은 곧 의미가 아닙니다. 0–1023 raw 값을 기준선, 변화량, 상태 이름으로 바꾸는 연습이 인터랙티브 작품의 핵심입니다.
강사 설명

A0~A5는 전압을 숫자로 읽는 입력이고, 센서값은 환경마다 달라서 현장 calibration이 필요하다고 설명합니다.

학생 활동

B4 pot, I1 CDS, I2 NTC, I3 joystick을 돌려 raw→normalized→state 변환을 봅니다.

Evidence

raw min/max, threshold, normalized value, joystick center/deadzone 기록.

넘어가기 조건

학생이 “raw 값 그대로 쓰면 왜 위험한지”와 “threshold를 현장에서 다시 잡아야 하는 이유”를 설명합니다.

2:00–2:45

거리·원격 입력

HC-SR04와 IR remote를 통해 zone, key map, mode switching을 구성합니다.

왜 이 구간이 필요한가요? 거리와 리모컨은 숫자 입력을 작품 제어 언어로 바꾸는 대표 사례입니다. “30cm 이하 = near”, “OK 버튼 = mode change”처럼 입력을 의미로 번역합니다.
강사 설명

센서값은 흔들리므로 zone과 timeout이 필요하고, 리모컨 코드는 리모컨마다 다를 수 있다고 설명합니다.

학생 활동

거리값을 near/mid/far로 나누고, IR 버튼 코드를 표로 적어 mode 전환에 연결합니다.

Evidence

distance_cm, zone, ir_code, mode JSON field.

넘어가기 조건

손 위치가 바뀔 때 zone이 안정적으로 바뀌고, 버튼 하나가 mode를 바꾸면 통과입니다.

2:45–3:35

I2C와 표시

I2C scanner로 LCD/OLED/RTC/MPU6050 주소를 찾고, display dashboard를 만듭니다.

왜 I2C를 배워야 하나요? 여러 모듈이 SDA/SCL 두 선을 공유할 수 있다는 감각이 생기면, 복잡한 센서/표시장치도 “주소가 다른 장치들”로 이해할 수 있습니다.
강사 설명

I2C는 버스이고, 주소 발견은 시작일 뿐 값이 정상인지 확인해야 PASS라고 설명합니다.

학생 활동

scanner로 주소를 찾고 LCD/OLED에 sensor raw 또는 state를 표시합니다.

Evidence

I2C address list, display text, RTC/MPU raw read.

넘어가기 조건

주소 발견 + 실제 값 읽기 + 표시장치 출력이 모두 확인되면 통과입니다.

3:35–4:25

표시·출력 확장

74HC595, MAX7219로 핀 확장과 driver 개념을 익힙니다.

왜 확장을 배우나요? Arduino pin은 한정되어 있고, 표시장치는 여러 LED를 빠르게 제어해야 합니다. shift register와 driver IC는 “핀 부족을 해결하는 방식”을 보여줍니다.
강사 설명

data/clock/latch와 driver/multiplexing의 역할을 생활 비유로 설명합니다.

학생 활동

74HC595 LED bar와 MAX7219 counter/icon 표시를 실행합니다.

Evidence

bit pattern, count value, display update interval.

넘어가기 조건

학생이 “Arduino가 모든 LED를 직접 들고 있지 않아도 되는 이유”를 설명하면 통과입니다.

왜 안전 경계를 따로 잡나요? 구동부는 초보자가 가장 쉽게 과장하거나 위험하게 연결하는 구간입니다. Arduino pin은 부하를 직접 먹여 살리는 전원이 아니라 명령선이라는 점을 반복합니다.
강사 설명

Beginner · B 시리즈디지털 출력/입력, 전류 제한, active-low, PWM의 기초
B0

키트 인벤토리와 개발환경

수업의 첫 실패 지점은 코드가 아니라 부품과 포트입니다. 학생이 UNO, USB, 브레드보드, 저항, 점퍼선을 직접 구분하게 합니다.

  • 핵심 개념: 보드/포트/업로드/Serial 창
  • 강사 질문: “이 케이블은 데이터 전송이 되나요?”

PASS: 학생이 자기 보드에 예제 sketch를 올리고 Serial 창을 열 수 있습니다.

B1

LED와 저항 신호등

LED는 가장 단순한 출력이지만 전류 제한과 GND return을 설명하는 첫 회로입니다. “켜졌다”보다 “전류가 어디로 돌아가는가”가 중요합니다.

  • 핵심 개념: 직렬 저항, 극성, 출력 pin
  • 강사 질문: “저항을 빼면 왜 안 되나요?”

PASS: D9→저항→LED→GND 경로를 말로 설명합니다.

B2

택트 스위치 active-low

버튼은 “누르면 1”이 아니라 INPUT_PULLUP에서는 “누르면 LOW/0”입니다. 이 어긋남을 이해하면 회로와 코드 약속을 분리해서 볼 수 있습니다.

  • 핵심 개념: floating, pull-up, active-low, debounce
  • 강사 질문: “왜 pressed인데 raw는 0인가요?”

PASS: raw와 pressed를 서로 다른 값으로 설명합니다.

B3

수동/능동 부저 비교

겉으로 비슷한 부품도 제어 방식이 다를 수 있음을 보여줍니다. 수동 부저는 주파수, 능동 부저는 on/off 명령이 핵심입니다.

  • 핵심 개념: tone frequency vs digital on/off
  • 강사 질문: “왜 하나는 멜로디가 되고 하나는 beep인가요?”

PASS: 수동/능동 부저의 제어 차이를 말합니다.

B4

  • 핵심 개념: analog input, mapping, PWM command
  • 강사 질문: “A0의 0–1023이 왜 D9의 0–255가 되나요?”

PASS: pot_raw, pot_norm, led_pwm이 함께 변하는 것을 확인합니다.

  • 강사 질문: “손을 떼었는데 왜 값이 정확히 512가 아닐까요?”

I4

HC-SR04 거리 구역

거리값을 그대로 쓰기보다 near/mid/far zone으로 바꾸어 작품 cue로 사용합니다. 물리 고정과 timeout 처리가 중요합니다.

  • 핵심 개념: trigger/echo, 거리 계산, timeout, zone
  • 강사 질문: “손이 없을 때는 어떤 값으로 처리해야 할까요?”

PASS: distance_cm과 zone이 안정적으로 바뀝니다.

I5

IR 리모컨 입력

리모컨은 버튼 이름보다 수신 code map이 중요합니다. 학생은 자기 리모컨의 코드표를 만들고 mode switching에 연결합니다.

  • 핵심 개념: code map, mode, repeat signal
  • 강사 질문: “내 리모컨의 OK 버튼 code는 남의 것과 같을까요?”

PASS: 수신 code를 표로 남기고 mode 전환에 연결합니다.

const int BUTTON_PIN = 2;
const int LED_PIN = 9;
unsigned long frame = 0;
int lastStable = HIGH;
int lastReading = HIGH;
unsigned long lastChangeMs = 0;
const unsigned long DEBOUNCE_MS = 30;

void setup() {
  pinMode(BUTTON_PIN, INPUT_PULLUP);
  pinMode(LED_PIN, OUTPUT);
  Serial.begin(115200);
}

void loop() {
  int reading = digitalRead(BUTTON_PIN);
  if (reading != lastReading) {
    lastChangeMs = millis();
    lastReading = reading;
  }
  if (millis() - lastChangeMs > DEBOUNCE_MS) {
    lastStable = reading;
  }
  int pressed = (lastStable == LOW) ? 1 : 0;
  digitalWrite(LED_PIN, pressed ? HIGH : LOW);

  Serial.print("{\"lab\":\"B2\",\"frame\":"); Serial.print(frame++);
  Serial.print(",\"ms\":"); Serial.print(millis());
  Serial.print(",\"raw\":"); Serial.print(lastStable == LOW ? 0 : 1);
  Serial.print(",\"pressed\":"); Serial.print(pressed);
  Serial.print(",\"event\":\""); Serial.print(pressed ? "pressed" : "released");
  Serial.print("\",\"valid\":1}");
  Serial.write('\n');
  delay(20);
}

unsigned long frame = 0;
int baseline = 0;
int samples = 0;

void setup() {

void loop() {

const int CLK_PIN = 13;
const int CS_PIN = 10;

int rowIndex = 1;

void maxSend(byte address, byte value) {

  shiftOut(DIN_PIN, CLK_PIN, MSBFIRST, address);
  shiftOut(DIN_PIN, CLK_PIN, MSBFIRST, value);

이 문서는 키트 기반 아두이노 수업 자료를 실제 동작 가능한 방향으로 확장하기 위한 프롬프트 묶음입니다. 목표는 “센서가 값을 보낸다”에서 멈추지 않고, 다음 계층을 모두 닫는 것입니다.

  2. TouchDesigner는 Serial DAT에서 raw row를 먼저 확인하고, parser가 통과한 값만 CHOP/cue로 넘깁니다.
  3. AI agent가 만든 회로·코드·패치는 가설이며, compile/upload/log/실물 관찰 evidence가 있어야 PASS입니다.

6.4 TouchDesigner parser 예시

아래 코드는 DAT Execute DAT에서 serial1의 table change를 감시해 마지막 row를 읽고, td_channels Table DAT와 td_status Text DAT를 갱신하는 예시입니다. 실제 callback 함수명은 DAT Execute DAT 설정에 맞춰 사용합니다.

# DAT Execute DAT example: monitor serial1 table changes.
# Operators expected:
# - serial1: Serial DAT receiving one JSON object per line
# - td_channels: Table DAT for parsed numeric values
# - td_status: Text DAT for parser status

import json
import time

STALE_SECONDS = 0.5

def _write_status(message):
    status = op('td_status')
    if status is not None:
        status.text = message


def _write_channels(msg, stale=0):
    table = op('td_channels')
    if table is None:
        return

    pot_norm = float(msg.get('pot_norm', 0.0))
    light_norm = float(msg.get('light_norm', 0.0))
    button = int(msg.get('button', 0))
    led_pwm_norm = float(msg.get('led_pwm', 0)) / 255.0
    valid = int(msg.get('valid', 0))

    table.clear()
    table.appendRow(['name', 'value'])
    table.appendRow(['pot_norm', pot_norm])
    table.appendRow(['light_norm', light_norm])
    table.appendRow(['button', button])
    table.appendRow(['led_pwm_norm', led_pwm_norm])
    table.appendRow(['valid', valid])
    table.appendRow(['stale', int(stale)])
    table.appendRow(['frame', int(msg.get('frame', -1))])


def parse_latest_serial_row():
    src = op('serial1')
    if src is None or src.numRows < 1:
        _write_status('WAITING: serial1 has no rows yet')
        return

    raw = src[src.numRows - 1, 0].val.strip()
    if not raw:
        _write_status('IGNORED: empty serial row')
        return

    try:
        msg = json.loads(raw)
    except Exception as e:
        _write_status('INVALID JSON: {} | raw={}'.format(e, raw[:160]))
        return

    required = ['frame', 'ms', 'pot_norm', 'light_norm', 'button', 'led_pwm', 'valid']
    missing = [key for key in required if key not in msg]
    if missing:
        _write_status('INVALID SCHEMA: missing {}'.format(', '.join(missing)))
        return

    _write_channels(msg, stale=0)
    _write_status('OK frame={} ms={}'.format(msg.get('frame'), msg.get('ms')))


# Enable “Table Change” on the DAT Execute DAT and call this function.
def onTableChange(dat):
    parse_latest_serial_row()
    return
