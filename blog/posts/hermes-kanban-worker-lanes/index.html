Hermes Kanban worker lanesлҠ” мһ‘м—…мһҗлҘј н”„лЎңм„ёмҠӨлЎң л§Ңл“ лӢӨ

Explains Hermes Agent kanban worker lanes: assignees, spawned profile processes, lifecycle termination, logs, and review handoffs.

Hermes Kanbanмқ„ лӢЁмҲңн•ң н•  мқј лӘ©лЎқмңјлЎң ліҙл©ҙ м Ҳл°ҳл§Ң мқҙн•ҙн•ң кІғмқҙлӢӨ. н•өмӢ¬мқҖ м№ҙл“ңк°Җ м•„лӢҲлқј worker laneмқҙлӢӨ. worker laneмқҖ вҖңмқҙ мһ‘м—…мқ„ м–ҙл–Ө мӢӨн–үмһҗм—җкІҢ л§Ўкёё кІғмқёк°ҖвҖқлҘј м •н•ҳкі , к·ё мӢӨн–үмһҗк°Җ лі„лҸ„ н”„лЎңм„ёмҠӨлЎң к№Ём–ҙлӮҳ мһ‘м—…мқ„ мІҳлҰ¬н•ң л’Ө, кІ°кіјлҘј лӢӨмӢң ліҙл“ңм—җ лӮЁкё°кІҢ н•ҳлҠ” кі„м•ҪмқҙлӢӨ. мқҙ кё°лҠҘмқҖ кёҙ мһ‘м—…мқ„ н•ң лҢҖнҷ” м•Ҳм—җ л°Җм–ҙ л„Јм§Җ м•Ҡкі , м—ӯн• В·лЎңк·ёВ·мһ¬мӢңлҸ„ к°ҖлҠҘн•ң мһ‘м—… нқҗлҰ„мңјлЎң л°”кҫёлҠ” л¬ём ңлҘј н•ҙкІ°н•ңлӢӨ.

кё°лҠҘ к°ңмҡ”: lane, assignee, lifecycle

Kanban ліҙл“ңлҠ” мһ‘м—…мқҳ м§„мӢӨмқ„ мҶҢмң н•ңлӢӨ. м№ҙл“ң мғҒнғңлҠ” triage, todo, ready, running, blocked, done, archived к°ҷмқҖ lifecycleлЎң мӣҖм§Ғмқҙкі , worker laneмқҖ к·ёмӨ‘ н•ҳлӮҳмқҳ м№ҙл“ңлҘј мӢӨн–үн•ҳлҠ” кө¬нҳ„мІҙлӢӨ. кё°ліё lane нҳ•нғңлҠ” Hermes profile laneмқҙлӢӨ. м№ҙл“ңмқҳ assigneeк°Җ н”„лЎңн•„ мқҙлҰ„кіј л§һмңјл©ҙ dispatcherк°Җ н•ҙлӢ№ н”„лЎңн•„мқ„ мғҲ Hermes н”„лЎңм„ёмҠӨлЎң мӢӨн–үн•ңлӢӨ.

мӨ‘мҡ”н•ң кө¬л¶„мқҖ мқҙкІғмқҙлӢӨ. workerлҠ” мқјмқ„ н•ҳм§Җл§Ң, мҷ„лЈҢ нҢҗм •мқҖ ліҙл“ңм—җ кё°лЎқлҗңлӢӨ. м •мғҒ мў…лЈҢлҠ” kanban_complete(summary=..., metadata=...)мқҙкі , мӮ¬лһҢмқҳ нҷ•мқёмқҙ н•„мҡ”н•ҳл©ҙ kanban_block(reason=...)мңјлЎң л©Ҳм¶ҳлӢӨ. м•„л¬ҙ terminator м—Ҷмқҙ н”„лЎңм„ёмҠӨк°Җ лҒқлӮҳл©ҙ м„ұкіөмқҙ м•„лӢҲлқј crash/gave-up кі„м—ҙмқҳ мӢӨнҢЁлЎң лӢӨлӨ„м§„лӢӨ.

м–ҙл–»кІҢ мһ‘лҸҷн•ҳлӮҳ

мӮ¬лһҢмқҙлӮҳ мһҗлҸҷнҷ”лҠ” CLIлЎң м№ҙл“ңлҘј л§Ңл“ лӢӨ.

hermes kanban create "Fix renderer regression" --assignee dev --workspace worktree
hermes kanban list
hermes kanban show <task-id>

dispatcherлҠ” мЈјкё°м ҒмңјлЎң ready мһ‘м—…мқ„ м°ҫм•„ claimн•ҳкі , assigneeм—җ л§һлҠ” laneмқ„ spawnн•ңлӢӨ. мҲҳлҸҷмңјлЎң н•ң лІҲл§Ң нҷ•мқён•  л•ҢлҠ” лӢӨмқҢмІҳлҹј dry-runмқ„ ліј мҲҳ мһҲлӢӨ.

hermes kanban dispatch --dry-run --max 3

н”„лЎңн•„ laneмқҙ мӢӨн–үлҗ  л•Ң worker н”„лЎңм„ёмҠӨм—җлҠ” мһ‘м—… id, ліҙл“ң DB, ліҙл“ң slug, workspace, run id, claim lock к°ҷмқҖ нҷҳкІҪк°’мқҙ мЈјмһ…лҗңлӢӨ. workerлҠ” м…ё лӘ…л №мңјлЎң ліҙл“ңлҘј мЎ°мһ‘н•ҳлҠ” лҢҖмӢ  kanban_show, kanban_comment, kanban_complete, kanban_block, kanban_heartbeat к°ҷмқҖ м „мҡ© лҸ„кө¬лҘј нҶөн•ҙ к°ҷмқҖ SQLite ліҙл“ңм—җ кё°лЎқн•ңлӢӨ. л”°лқјм„ң лҢ“кёҖ, мқҙлІӨнҠё, мӢӨн–ү мқҙл Ҙ, лЎңк·ёк°Җ н•ң task мЈјліҖм—җ лӘЁмқёлӢӨ.

кІҖнҶ к°Җ н•„мҡ”н•ң мҪ”л“ң мһ‘м—…м—җм„ңлҠ” л°”лЎң мҷ„лЈҢн•ҳм§Җ м•ҠлҠ” кҙҖлЎҖк°Җ мң мҡ©н•ҳлӢӨ. лЁјм Җ commentм—җ changed_files, tests_run, diff_path к°ҷмқҖ кө¬мЎ°нҷ” м •ліҙлҘј лӮЁкё°кі , review-required:лЎң мӢңмһ‘н•ҳлҠ” block reasonмқ„ лӮЁкёҙлӢӨ. лҰ¬л·°м–ҙк°Җ мҠ№мқён•ҙ unblockн•ҳл©ҙ к°ҷмқҖ м№ҙл“ңк°Җ лӢӨмӢң мӢӨн–үлҗҳм–ҙ нӣ„мҶҚ мһ‘м—…мқ„ мқҙм–ҙк°„лӢӨ.

мӢӨм ң мӮ¬мҡ© мӮ¬мқҙл“ңл°”

мқҙ мӮ¬мҡ©мһҗмқҳ мҡҙмҳҒм—җм„ңлҠ” м§ҖмӢқ м •лҰ¬, к°ңл°ң, мҡҙмҳҒ нҷ•мқёмқ„ м„ңлЎң лӢӨлҘё н”„лЎңн•„лЎң лӮҳлҲ„кі , кёҙ мһ‘м—…мқҖ м№ҙл“ңмҷҖ worker laneмңјлЎң л„ҳкёҙлӢӨ. нҠ№нһҲ к°ңл°ң мһ‘м—…мқҖ н•ң н”„лЎңн•„мқҙ кі„мҶҚ л– м•Ҳкё°ліҙлӢӨ inspect, implement, test, review м„ұкІ©мқ„ л¶„лҰ¬н•ҙ лЎңк·ёмҷҖ artifactлҘј лӮЁкё°кІҢ н•ңлӢӨ. мқҙ мӮ¬лЎҖм—җм„ң мӨ‘мҡ”н•ң м җмқҖ н”„лЎңн•„ мқҙлҰ„ мһҗмІҙк°Җ м•„лӢҲлқј, к°Ғ laneмқҙ м–ҙл–Ө мў…лЈҢ мЎ°кұҙкіј мҰқкұ°лҘј лӮЁкё°лҠ”к°ҖлӢӨ.

н•Ём •кіј нҢҒ

  • delegate_taskмҷҖ нҳјлҸҷн•ҳм§Җ м•ҠлҠ”лӢӨ. delegationмқҖ л¶ҖлӘЁк°Җ кІ°кіјлҘј кё°лӢӨлҰ¬лҠ” н•ЁмҲҳ нҳём¶ңм—җ к°Җк№қкі , KanbanмқҖ мһ¬мӢңмһ‘кіј мӮ¬лһҢ к°ңмһ…мқҙ к°ҖлҠҘн•ң durable queueлӢӨ.
  • assignee мҳӨнғҖлҠ” мЎ°мҡ©нһҲ лӢӨлҘё workerм—җкІҢ л„ҳм–ҙк°Җм§Җ м•ҠлҠ”лӢӨ. мӨҖл№„ мғҒнғңм—җ лӮЁкі  diagnosticsлӮҳ skipped мқҙлІӨнҠёлЎң л“ңлҹ¬лӮҳлҜҖлЎң hermes kanban assigneesмҷҖ hermes profile listлҘј н•Ёк»ҳ нҷ•мқён•ңлӢӨ.
  • scratch workspaceлҠ” мҷ„лЈҢ мӢң мӮ¬лқјм§Ҳ мҲҳ мһҲлӢӨ. ліҙмЎҙн•  мӮ°м¶ңл¬јмқҙ мһҲмңјл©ҙ dir:<absolute-path> лҳҗлҠ” worktreeлҘј кі лҘёлӢӨ.
  • workerк°Җ лҒқлӮ¬лӢӨлҠ” л§җліҙлӢӨ log, run history, comment metadataк°Җ лҚ” мӨ‘мҡ”н•ҳлӢӨ. hermes kanban runs <task-id>мҷҖ hermes kanban tail <task-id>лҘј нҷ•мқён•ңлӢӨ.
  • gateway м•Ҳм—җм„ң dispatcherк°Җ лҸҢлҸ„лЎқ м„Өм •н•ң кІҪмҡ°, gateway мғҒнғңк°Җ кі§ worker lane мғҒнғңмқҳ м¶ңл°ңм җмқҙлӢӨ.

л§Ҳл¬ҙлҰ¬

worker laneмқҖ HermesлҘј вҖңлӢөліҖн•ҳлҠ” м—җмқҙм „нҠёвҖқм—җм„ң вҖңмһ‘м—…мқ„ л§Ўм•„ мӢӨн–үн•ҳкі  кё°лЎқн•ҳлҠ” н”„лЎңм„ёмҠӨ м§‘н•©вҖқмңјлЎң нҷ•мһҘн•ңлӢӨ. м§§мқҖ мЎ°мӮ¬лӮҳ н•ң лІҲмқҳ нҢҗлӢЁмқҖ delegationмқҙ к°ҖліҚлӢӨ. н•ҳм§Җл§Ң м—ӯн• мқҙ лӮҳлүҳкі , мһ¬мӢңлҸ„мҷҖ лҰ¬л·°к°Җ н•„мҡ”н•ҳкі , лӮҙмқјлҸ„ к°ҷмқҖ кё°лЎқмқ„ лӢӨмӢң лҙҗм•ј н•ңлӢӨл©ҙ Kanban worker laneмқҙ лҚ” м•Ҳм „н•ң м„ нғқмқҙлӢӨ.
