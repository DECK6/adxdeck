# GPT-5.5는 GPT-5.4를 아직 대체하지 않는다

OpenAI가 2026년 4월 23일 GPT-5.5를 공개했다. 발표 문구만 보면 결론은 간단해 보인다. 더 똑똑하고, 더 자율적이고, 더 적은 토큰으로 일을 끝낸다는 것이다. 그러면 GPT-5.4는 바로 이전 세대가 되었고, 우리는 이제 GPT-5.5만 보면 되는 걸까?

아직은 아니다. GPT-5.5는 분명히 GPT-5.4보다 강한 모델로 발표되었지만, 실제 선택지는 “5.5가 5.4를 이겼다”보다 복잡하다. 2026년 4월 24일 확인 기준으로 GPT-5.5는 ChatGPT와 Codex에는 제공되지만 API는 아직 “coming soon” 상태다. 반대로 GPT-5.4는 이미 ChatGPT, API, Codex에 모두 들어와 있고, 모델 문서와 가격, 최대 출력, 컨텍스트, 도구 지원 정보가 정리되어 있다.

![GPT-5.5 vs GPT-5.4 cover](images/gpt-5-5-vs-gpt-5-4-cover.png)

그래서 지금 필요한 질문은 “어느 쪽이 더 좋은가”가 아니라 “어떤 작업에서 GPT-5.5의 추가 비용과 불확실성을 감수할 만한가”다. 그 질문으로 보면 두 모델의 차이는 훨씬 선명해진다.

## 차이는 지식보다 작업 방식에 있다

GPT-5.4는 “전문 업무를 위한 프런티어 모델”로 공개되었다. OpenAI는 GPT-5.4를 reasoning, coding, agentic workflow, 스프레드시트, 프레젠테이션, 문서 작업을 하나로 묶는 모델로 설명했다. 특히 API와 Codex에서 1M 토큰급 긴 컨텍스트를 다루고, 컴퓨터 사용 능력과 도구 사용을 강화한 것이 핵심이었다.

GPT-5.5의 강조점은 여기서 한 단계 더 나간다. 공식 발표의 톤은 “더 정확한 답변”보다 “더 적은 관리로 더 긴 일을 끝내는 모델”에 가깝다. 코딩, 온라인 리서치, 데이터 분석, 문서와 스프레드시트 생성, 소프트웨어 조작, 여러 도구를 오가며 끝까지 진행하는 작업이 반복해서 언급된다.

즉 GPT-5.4가 전문 업무를 빠르고 정확하게 처리하는 범용 프런티어 모델이라면, GPT-5.5는 그중에서도 긴 맥락, 도구 사용, 검증, 모호성 처리, 반복 실행이 섞인 agentic work에 더 초점을 둔 모델이다. 차이는 단순 지식량보다 작업 지속성에 있다.

## 벤치마크는 “전면적 압도”보다 “에이전트형 작업 강화”를 말한다

GPT-5.5가 모든 지표에서 갑자기 압도적인 격차를 만든 것은 아니다. 예를 들어 SWE-Bench Pro는 GPT-5.4의 57.7%에서 GPT-5.5의 58.6%로 오른다. 이 수치만 보면 “코딩 성능이 완전히 다른 단계가 되었다”고 말하기는 어렵다.

하지만 Terminal-Bench 2.0에서는 75.1%에서 82.7%로 격차가 커진다. Expert-SWE 내부 평가도 68.5%에서 73.1%로 오른다. 이 두 지표는 단발 코드 작성보다 계획, 반복, 명령줄 사용, 도구 조율, 긴 문제 해결 흐름에 더 가깝다.

핵심 지표만 추리면 이렇게 볼 수 있다.

| 평가 영역 | GPT-5.4 | GPT-5.5 | 읽는 법 |
|---|---:|---:|---|
| SWE-Bench Pro | 57.7% | 58.6% | 일반적인 GitHub 이슈 해결은 소폭 개선 |
| Terminal-Bench 2.0 | 75.1% | 82.7% | CLI 기반 장기 작업과 도구 조율에서 개선 폭이 큼 |
| Expert-SWE | 68.5% | 73.1% | 긴 코딩 작업에서 개선 |
| GDPval | 83.0% | 84.9% | 전문 업무 품질은 소폭 개선 |
| OSWorld-Verified | 75.0% | 78.7% | 실제 컴퓨터 환경 조작 능력 개선 |
| BrowseComp | 82.7% | 84.4% | 브라우징 기반 복합 질의 개선 |
| FrontierMath Tier 4 | 27.1% | 35.4% | 어려운 수학 문제에서 더 큰 개선 |

이 표가 말하는 것은 단순한 세대교체가 아니다. GPT-5.5의 강점은 “질문 하나에 답을 잘하는 모델”보다 “복잡한 일을 맡겨두면 중간에 덜 멈추고 더 오래 밀고 가는 모델”에 가깝다. 특히 Codex 사용자에게 이 차이는 더 크게 느껴질 가능성이 높다.

## API에서는 아직 GPT-5.4가 기준 모델이다

개발자 입장에서는 이 부분이 가장 중요하다. OpenAI Developers 모델 문서는 GPT-5.5가 현재 ChatGPT와 Codex에서 사용 가능하며 API는 곧 제공될 예정이라고 적고 있다. 같은 문서는 시작 모델로 `gpt-5.4`를 권한다. 복잡한 reasoning과 coding에는 `gpt-5.4`, 비용과 지연 시간이 중요하면 `gpt-5.4-mini`나 `gpt-5.4-nano`를 고르라는 구조다.

GPT-5.4는 이미 API 모델 페이지가 있다. 확인 가능한 정보도 구체적이다. 모델 ID는 `gpt-5.4`, 컨텍스트 윈도우는 1,050,000 토큰, 최대 출력은 128,000 토큰, 지식 컷오프는 2025년 8월 31일이다. 함수 호출, 웹 검색, 파일 검색, 컴퓨터 사용도 지원 목록에 들어가 있다.

반면 GPT-5.5는 가격과 1M 컨텍스트가 공개되었지만, API의 세부 운영 정보는 아직 다 열리지 않았다. 모델 ID, 스냅샷, 최대 출력, rate limit, 도구 지원, 실제 SDK 예제까지 확인되기 전에는 프로덕션 기본값으로 삼기 어렵다.

그래서 오늘 당장 API 제품을 만들어야 한다면 답은 단순하다. 기본값은 GPT-5.4다. GPT-5.5는 API가 열린 뒤, 같은 작업셋으로 A/B 테스트를 걸어 판단하는 모델이다.

## 가격은 2배지만, 작업당 비용은 따로 봐야 한다

공식 가격표 기준으로 GPT-5.5의 예정 API 단가는 GPT-5.4의 정확히 2배다. GPT-5.5는 입력 100만 토큰당 5달러, 캐시 입력 0.50달러, 출력 30달러다. GPT-5.4는 입력 2.50달러, 캐시 입력 0.25달러, 출력 15달러다.

하지만 모델 가격은 토큰 단가만으로 끝나지 않는다. OpenAI는 GPT-5.5가 Codex에서 GPT-5.4보다 더 적은 토큰으로 더 나은 결과를 내도록 튜닝되었다고 설명한다. 이 말이 중요하다. 비싼 모델이 항상 비싼 작업을 만든다는 뜻은 아니기 때문이다.

실제 비교는 이렇게 해야 한다.

```text
작업당 비용 =
입력 토큰 비용
+ 캐시 입력 토큰 비용
+ 출력 토큰 비용
+ 재시도 비용
+ 사람이 수정하는 시간
```

짧은 요약, 간단한 분류, 대량 반복 처리, 낮은 위험도의 일반 Q&A라면 GPT-5.5는 과한 선택일 수 있다. 이 경우에는 GPT-5.4, GPT-5.4 mini, GPT-5.4 nano 쪽이 더 경제적이다. 반대로 긴 코드 수정, 실패 원인 추적, 다단계 리서치, 문서와 데이터와 도구 사용이 섞인 작업이라면 GPT-5.5가 더 적은 재시도와 더 적은 총 토큰으로 비용을 회수할 가능성이 있다.

다만 이 판단은 반드시 “내 작업”으로 측정해야 한다. GPT-5.5가 모든 작업에서 자동으로 싸지는 모델이라는 뜻은 아니다. 더 정확한 말은 “긴 에이전트형 작업에서는 명목 단가보다 작업당 완료 비용을 봐야 한다”다.

## 안전성과 접근 조건도 선택 기준이 된다

GPT-5.5는 성능만 올라간 모델이 아니다. OpenAI는 GPT-5.5의 biological/chemical 및 cybersecurity capabilities를 Preparedness Framework상 High로 취급한다고 밝혔다. Critical cybersecurity 수준에는 도달하지 않았지만, GPT-5.4보다 사이버 역량이 한 단계 높아졌다고 설명한다.

이 말은 보안 방어, 취약점 분석, 바이오·화학 관련 연구를 하는 사용자에게 양면적이다. 한편으로는 더 강한 방어 도구가 생긴다. 다른 한편으로는 더 촘촘한 분류기, 더 엄격한 검토, trusted access 같은 접근 조건이 중요해질 수 있다.

따라서 GPT-5.5를 단지 “더 좋은 GPT-5.4”로만 보면 놓치는 것이 있다. 이 모델은 능력이 올라간 만큼 운영 정책과 접근 조건도 더 중요한 모델이다. 특히 cyber, bio, chem 경계 작업에서는 성능뿐 아니라 허용 범위와 사용 맥락을 함께 설계해야 한다.

## 지금의 실무 결론

GPT-5.5는 GPT-5.4의 단순한 마이너 업데이트가 아니다. 특히 Codex, 긴 코딩 작업, 컴퓨터 사용, 다단계 지식 작업에서는 더 큰 차이를 만들 수 있는 모델이다. 발표 자료와 벤치마크를 종합하면 GPT-5.5의 핵심은 “더 똑똑한 답변”보다 “더 적은 지시로 더 긴 일을 맡는 능력”이다.

하지만 GPT-5.5가 GPT-5.4를 오늘 바로 대체한다고 말하기도 어렵다. API 기준으로는 아직 GPT-5.4가 확정 모델이고, GPT-5.5는 출시 예정 정보와 제품 배포 정보 사이에 걸쳐 있다. 가격도 2배이므로 모든 작업에 기본값으로 넣을 모델은 아니다.

현재 기준의 선택은 이렇게 정리할 수 있다.

| 상황 | 우선 선택 |
|---|---|
| 지금 API 프로덕션에 넣어야 한다 | GPT-5.4 |
| 비용과 지연 시간이 더 중요하다 | GPT-5.4 mini 또는 nano |
| Codex에서 긴 코드베이스 작업을 맡긴다 | GPT-5.5 테스트 가치 높음 |
| 온라인 리서치, 문서, 데이터, 도구 사용이 섞인다 | GPT-5.5 테스트 가치 높음 |
| 짧은 답변, 단순 분류, 대량 처리 | GPT-5.5는 과할 수 있음 |
| cyber/bio/chem 경계 작업 | 성능보다 정책·접근 조건까지 함께 검토 |

결론은 간단하다. GPT-5.5는 “무조건 갈아탈 모델”이 아니라 “비싼 만큼 긴 일을 더 잘 끝내는지 검증해야 할 모델”이다. GPT-5.4는 여전히 현재 API의 기준점이고, GPT-5.5는 그 기준점 위에서 에이전트형 업무의 상한을 다시 밀어 올리는 모델이다.
