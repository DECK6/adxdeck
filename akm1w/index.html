01 / WHY AKM

왜 AKM인가

노트가 많다는 사실만으로 AI가 원본, 현재 결정, 적용 범위, 실행 절차를 구분하지는 못합니다. AKM은 기존 지식의 집을 허무는 앱이 아니라, 에이전트가 그 집의 방과 규칙을 읽는 운영 지도입니다.

BEFORE / 자주 생기는 장면

자료는 있는데 매번 다시 설명합니다.

  • 원본 링크, 개인 해석, 프로젝트 결정, 체크리스트, 실패 기록이 한곳에 섞입니다.
  • 좋은 답변은 대화에 묻히고, 오래된 초안이 현재 기준처럼 다시 사용됩니다.
  • 검색 결과가 나와도 그것이 권위 있는 최신 기준인지 다시 확인해야 합니다.
AFTER / 책임을 분리한 상태

같은 내용을 서로 다른 책임으로 나눕니다.

  • 원본은 Source로 보존하고, 재사용 판단은 Knowledge로 합성합니다.
  • 프로젝트 조건은 Context, 반복 단계와 stop rule은 Procedure로 둡니다.
  • 실패를 Evaluation에서 판정하고 수정할 레이어로 Learn Back합니다.

Obsidian, Notion, Logseq, 폴더를 그대로 유지합니다. 기존 위치를 가리키는 포인터와 작은 manifest부터 시작합니다. 첫 주에 전체 Vault를 옮기지 않습니다.

TWO KNOWLEDGE ZONES
답변 참고 지식

Source → Knowledge
사실 · 근거 · 정본 · 충돌

에이전트 운영 지식

Context + Operational Memory + Procedure
→ Action → Evaluation → Learn Back

답변 참고 지식은 “무엇을 말할 것인가”를 담당합니다. 에이전트 운영 지식은 “누구의 어떤 상황에서 어떤 순서와 제한으로 실행하고 검증할 것인가”를 담당합니다. Context는 답변과 무관한 격리 구역이 아닙니다. 사용자와 프로젝트 상황에 따라 답변의 적용 범위와 강조점을 바꿀 수 있습니다. Operational Memory는 매번 먼저 알아야 하는 짧고 안정적인 포인터이며, 긴 문서와 일시적 상태를 모두 넣는 곳이 아닙니다.

SECONDARY CASE / CEREBRAS

Cerebras는 Slack, 코드, 문서, 내부 시스템을 한곳으로 강제 이관하기보다 기존 출처에서 정보를 모아 사람·자동화·AI 에이전트가 함께 쓰는 검색 계층을 구축했다고 공개했습니다. 하루 15,000건 이상의 질의는 Cerebras의 자체 보고이며 독립 검증되지 않았습니다. 이 숫자는 보편 권장값이 아니라 “자료 위치”보다 “근거와 권한의 연결”이 중요하다는 보조 사례로만 읽습니다.

02 / SAFE INSTALL

안전한 별도 경로에서 시작합니다.

기존 개인 Vault, Notion export 폴더, 운영 Git 저장소, 이미 쓰는 akm 폴더 안에서 실행하지 않습니다. 설치가 막혀도 구조와 노트 실습은 계속하되 설치 성공을 주장하지 않습니다.

BEFORE CLASS / 수업 전 준비
  • Git 설치 여부를 확인합니다.
  • 새로운 빈 실습 폴더를 만들 수 있는 컴퓨터를 준비합니다.
  • 원본·기록 노트, 해석·요약 노트, 적용·프로젝트 노트를 한 개씩 고릅니다.
  • 기존 PKM을 쓴다면 현재 구조를 짧게 설명할 준비를 합니다.
  • 기존 Vault나 지식관리 시스템은 이동하거나 삭제하지 않습니다.
  • 설치가 막히면 privacy-safe 공개 샘플과 읽기 전용 fallback으로 노트 실습을 계속합니다.

먼저 자료를 확인합니다.

하나라도 확실하지 않으면 개인 노트를 사용하지 않습니다. token, proxy secret, 개인 credential로 회사 정책을 우회하지 않습니다.

4-MINUTE BLOCKER STOP RULE

네트워크·권한 문제를 오래 끌지 않습니다.

  1. Git 미설치, 네트워크 차단, 권한 문제가 4분 안에 풀리지 않으면 troubleshooting을 멈춥니다.
  2. 검증된 읽기 전용 fallback 또는 pair-observer 방식으로 구조·노트 실습을 계속합니다.
  3. 직접 설치하지 않았다면 NOT TESTED, 정책·권한 때문에 멈췄다면 HOLD입니다.
  4. fallback은 설치 성공 evidence가 아닙니다. Node.js와 lint도 첫 실습의 필수 조건이 아닙니다.

examples/minimal-akm/에서 Source와 Knowledge 예시를 나란히 읽습니다. 검색·lint·AI 답변을 원본 대신 사용하지 않습니다. 검색 점수와 lint 통과는 후보 찾기와 형식 점검이지, truth나 사실 검증이 아닙니다.

04 / CHOOSE LANE

깊이는 달라도 마지막 검증은 같습니다.

Advanced는 더 많이 쓰는 레인이 아닙니다. authority, freshness, scope 충돌을 더 엄격히 다루는 레인입니다. 모든 레인은 direct-read Verify와 Exit Ticket에서 다시 만납니다.

CURRENT LANE / BEGINNER

공개 샘플로 Source와 Knowledge를 나누고, 답변 노트 하나와 운영 카드 하나를 만듭니다.

출처 세 개와 결론 한 개를 연결하고, 운영 카드에 사용 시점과 stop rule을 한 줄씩 쓰면 통과합니다.

Advanced 필수 확인 필드: source · authority · scope · effective date · supersedes · reviewer · evidence · stop rule. 최신 파일이라는 이유만으로 정본을 고르지 않습니다.
