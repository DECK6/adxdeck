AI 트렌드 & 리서치 실무 6시간 실습 프롬프트 예시집

이 문서는 6시간 실습 수업을 따라가며 수강자가 바로 복사해 쓰고, 자신의 업무에 맞게 고쳐 쓸 수 있도록 만든 프롬프트 예시집이다. 원본 교안의 흐름을 기준으로 구성했지만, 장표 제작 메모나 화면 디자인 지시는 포함하지 않았다. 모든 예시는 실제 업무에 투입하기 전 사람 검수와 조직의 승인 절차를 전제로 한다.

사용 방법

  • 대괄호 [ ] 안의 값은 자신의 상황에 맞게 바꾼다.
  • 실제 고객명, 연락처, 내부 매출, 계약 정보, 민감정보는 넣지 않는다.
  • 자료 기반 작업에서는 “자료에 있는 사실 / 합리적 추정 / 제안”을 반드시 분리한다.
  • 생성된 결과물은 바로 발송하거나 게시하지 말고 검수 체크리스트를 통과시킨다.
  • 같은 프롬프트를 한 번에 완성하려 하지 말고, 초안 → 검토 → 수정 지시 → 최종 정리 순서로 반복한다.

공통 입력 변수 사전

  • [회사/조직]: 예) 커머스 브랜드, 교육기관, B2B SaaS 기업
  • [업무영역]: 예) 마케팅, 영업, 고객지원, 교육기획, 리서치
  • [제품/서비스]: 분석하거나 홍보할 제품·서비스명
  • [대상고객]: 예) 신규 고객, 기존 고객, 이탈 위험 고객, B2B 의사결정자
  • [자료]: 제품 소개서, 고객 후기, 경쟁사 문구, 시장 리포트, 내부 가이드 등
  • [채널]: 이메일, SMS, 카카오톡, SNS, 블로그, 랜딩페이지, 발표자료
  • [브랜드톤]: 예) 전문적, 친근한, 차분한, 프리미엄, 실용적
  • [금지조건]: 과장 표현, 확인되지 않은 수치, 경쟁사 비방, 민감정보 추정 등
  • [출력형식]: bullet, 섹션형 보고서, 체크리스트, 표 형식 목록, 이메일 문안 등

1. 오리엔테이션: 내 업무를 AI 흐름으로 바꾸기

Prompt 01#

프롬프트 01. 반복 업무 분해 캔버스

  • 목적: 내 업무를 AI에게 맡길 수 있는 입력, 처리, 검수, 산출물 구조로 나눈다.
  • 언제 사용하나: 수업 초반에 오늘 실습 주제를 정하거나, 현업 적용 대상을 고를 때 사용한다.
당신은 [업무영역] 업무 프로세스 컨설턴트입니다.

아래 업무를 AI 활용 관점에서 분해해 주세요.

업무 설명:
[반복 업무 설명]

회사/조직 맥락:
- 조직: [회사/조직]
- 제품/서비스: [제품/서비스]
- 대상고객: [대상고객]
- 현재 사용 자료: [자료]
- 최종 사용 채널: [채널]

다음 형식으로 정리해 주세요.
1. 업무 목표 1문장
2. 입력 자료 목록
3. AI가 처리하기 좋은 작업
4. 사람이 반드시 판단해야 할 작업
5. 최종 산출물
6. 검수 기준 5개
7. AI 사용 전 제거해야 할 민감정보
8. 오늘 수업에서 만들 수 있는 미니 산출물 1개

주의:
- 고객 개인정보, 내부 기밀, 확인되지 않은 수치는 사용하지 않는 전제로 작성하세요.
- 자동 발송이나 자동 게시가 아니라 사람 검수 후 사용하는 초안으로 설계하세요.
  • 입력 변수: [반복 업무 설명], [회사/조직], [제품/서비스], [대상고객], [자료], [채널]
  • 기대 출력: 업무 목표, 입력 자료, AI 처리, 사람 검수, 산출물, 민감정보 제거 기준이 한 번에 정리된 캔버스
Prompt 02#

프롬프트 02. 나쁜 요청을 좋은 업무 지시문으로 바꾸기

  • 목적: 단문 질문을 실무형 프롬프트로 확장한다.
  • 언제 사용하나: 프롬프트 엔지니어링 파트에서 나쁜 예시를 개선할 때 사용한다.
아래의 짧고 모호한 요청을 실무에서 사용할 수 있는 구조화 프롬프트로 개선해 주세요.

나쁜 요청:
[예: 우리 제품 홍보 문구 써줘]

반드시 포함할 요소:
- 역할: AI가 어떤 관점으로 일해야 하는가
- 목표: 최종적으로 무엇을 만들어야 하는가
- 맥락: [회사/조직], [제품/서비스], [대상고객], [채널]
- 입력 자료: [자료]
- 작업 절차: 요약 → 인사이트 → 초안 → 검수 순서
- 출력 형식: [출력형식]
- 제약조건: [금지조건]
- 검수 기준: 사실성, 근거, 톤, 과장 여부, 민감표현 여부

1. 개선된 프롬프트 전문
2. 추가된 조건 설명
3. 아직 필요한 자료 또는 질문
4. 실무 투입 전 검수 체크리스트
  • 입력 변수: [나쁜 요청], [회사/조직], [제품/서비스], [대상고객], [채널], [자료], [금지조건]
  • 기대 출력: 복사해서 사용할 수 있는 긴 업무 지시문과 누락 조건 목록

2. AI 기초와 거버넌스: 사실, 추정, 제안을 분리하기

Prompt 03#

프롬프트 03. AI 결과물 환각 점검

  • 목적: AI가 만든 답변에서 근거 없는 주장, 확인되지 않은 수치, 과장 표현을 찾아낸다.
  • 언제 사용하나: 리서치 요약, 보고서 초안, 메시지 초안을 받은 직후 사용한다.

검수 대상 초안:
[AI가 생성한 결과물]

참고 가능한 근거 자료:
[자료 또는 링크 요약]

검수 기준:
1. 근거 자료에 명시된 사실
2. 근거는 없지만 합리적으로 보이는 추정
3. 확인이 필요한 주장
4. 삭제하거나 완화해야 할 과장 표현
5. 법적·윤리적·브랜드 리스크가 있는 표현

출력 형식:
- 안전하게 사용 가능: [문장 또는 항목]
- 확인 필요: [문장 또는 항목 + 필요한 확인 자료]
- 수정 권장: [원문 → 수정안]
- 사용 금지: [이유]
- 최종 승인 전 질문 5개

주의:
- 모르면 모른다고 표시하세요.
- 근거 자료에 없는 내용을 사실처럼 확정하지 마세요.
  • 입력 변수: [AI가 생성한 결과물], [자료 또는 링크 요약]
  • 기대 출력: 사실, 추정, 확인 필요, 수정 권장, 사용 금지 항목이 분리된 검수 결과

자료 설명:
[자료]

3. 고객 동의 또는 사용 목적 범위 문제
